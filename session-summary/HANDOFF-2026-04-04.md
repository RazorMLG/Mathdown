## Failed Approaches (mandatory)

- **`line++` added to UL/OL cases after inner while** — caused list items to appear twice (double-rendered). Reverted. The inner while already advances `line` past all list items; no outer increment needed.
- **Swapping `currentLine = contentArr[line]` to before `line++` inside inner while** — caused first list item to be pushed twice. Reverted to original order (`line++` then `currentLine = ...`).
- **Unconditional `line++` in default case** — skipped the line after a paragraph when the while loop ran. Fixed by making it conditional: `if (textInParagraph === "") line++`.
- **Copy-paste of test markdown from rendered chat** — `>` became unicode `▎`, `#` and `` ` `` were consumed by chat rendering. All blocks appeared as paragraphs. Diagnosed by hardcoding a known-good string in `App.tsx` (reverted after confirmation). Always paste from raw source.

## Key Decisions

| Decision                                                  | Rationale                                                                                          |
| --------------------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| No `parser.ts`                                            | Tokenizer already produces a tree (`children` on every block token); separate AST step unnecessary |
| `switch(true)` in `parseBlocks`                           | Simple, readable, matches Jovan's level; switch cases evaluated in order — ORDER MATTERS           |
| Discriminated union `TokenShapes`                         | Better than generic `ASTNode` for TypeScript exhaustive checking                                   |
| `ListItemToken` type exported separately                  | `UnorderedListToken.children` typed as `ListItemToken[]`, not `TokenShapes[]`                      |
| Mutual recursion for renderer                             | `renderNode` → `renderChildren` → `renderNode`; call stack is the stack                            |
| `render()` exported as thin wrapper over `renderChildren` | Public API name separate from implementation detail                                                |
| Vitest unit tests over Playwright                         | Parser pipeline is pure TS, no DOM; Vitest is faster to set up for "quick e2e"                     |
